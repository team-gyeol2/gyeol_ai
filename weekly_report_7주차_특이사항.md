# 8. 특이사항 및 보고사항

**1. 피드백 반영 확인**
지난주 피드백을 반영하여 통신 장애 대응 방식을 물리적 위치 재조정 단독 적용에서, 통신 토폴로지 재구성(중계 노드 전환) → 물리적 위치 보정의 2단계 결합 구조로 수정하였습니다. "어떤 드론과 통신할 것인지의 연결 구조를 동적으로 재구성한다"는 통신 토폴로지 개념을 1단계 즉각 대응으로 적용하고, 이후에도 연결이 복원되지 않을 경우에만 물리적 이동을 수행하는 방식으로 설계하였습니다. 교수님께서 의도하신 방향이 이와 같은지 확인을 부탁드립니다.

**2. 리드타임 정의**
Early Warning 구현 시 슬라이딩 윈도우 레이블을 미래 시점 기준으로 재설정해야 할 것으로 판단됩니다. 실제 UAV 운용에서 적절한 리드타임 기준(예: 1초/3초/5초)이 어느 수준인지 여쭤보고 싶습니다.

**3. 시뮬레이션 데이터 한계**
현재 데이터셋은 자유공간 경로손실 모델 기반으로 생성되어 멀티패스, 도플러 효과 등 실제 환경 요소가 배제되어 있습니다. 이러한 시뮬레이션 데이터로 학습한 모델이 실제 환경에서도 유효할 수 있는지 의견을 여쭤보고 싶습니다.

**4. 물리적 위치 보정 현실성**
현재 알고리즘은 드론의 관성과 제어 지연을 고려하지 않은 단순 이동 방식입니다. 실제 드론 제어 관점에서 반드시 반영해야 할 제약 조건이 있는지 여쭤보고 싶습니다.
